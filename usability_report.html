Usability Test on a Website – Full Project Report

Abstract

Usability testing plays a pivotal role in ensuring digital products are user-friendly and accessible. This report presents a structured usability test performed on a sample e-commerce website using a combination of heuristic evaluation, task analysis, and direct user feedback. The goal was to uncover design inefficiencies, navigation challenges, and performance bottlenecks through real-world user interaction. The study also introduces an embedded feedback form using web technologies (HTML, CSS, JavaScript) to capture ongoing user insights in real-time. Based on the test data, actionable improvements are proposed to optimize the user journey, enhance accessibility, and increase task success rates across devices.

Introduction

The growing reliance on digital platforms requires that websites not only function correctly but also offer a seamless user experience. Usability is a measure of how easily and efficiently users can navigate and interact with a system. Despite advancements in front-end frameworks and UI libraries, many websites still fall short in terms of user-centered design. This study addresses this issue by assessing usability flaws in a live website and recommending targeted improvements.

Usability testing identifies how real users interact with a website to uncover points of confusion, friction, or inefficiency. By focusing on an e-commerce use case, this project reflects typical challenges in real-world online retail scenarios—such as long checkouts, unclear calls-to-action, and mobile responsiveness.

Objectives

  • To evaluate how intuitive and efficient the user interface is.
  • To identify barriers that prevent users from completing core tasks.
  • To implement a simple yet effective feedback system using web technologies.
  • To enhance user satisfaction through measurable interface improvements.
  • To propose recommendations based on both qualitative and quantitative data.

Scope and Limitations

Scope: This project includes desktop and mobile usability testing for a functional e-commerce website. Core flows such as browsing products, adding items to the cart, and completing checkout were assessed. It also includes the integration of a feedback module for continuous input.

Limitations: The sample size was limited to 15 users due to time constraints. Additionally, the feedback system relies on voluntary user input, which may not represent all usability issues. Technical testing such as load balancing and backend performance was not within the project's scope.

Literature Review

Summary of Existing Research

Nielsen (1994) introduced 10 usability heuristics which are widely adopted across the design industry. These include visibility of system status, user control, consistency, error prevention, and help documentation. Krug (2014) emphasized minimal thinking by users, focusing on clarity and directness.

Gaps in Current Research

  • Limited implementation of real-time feedback mechanisms during testing.
  • Infrequent integration of user sentiment alongside analytics metrics.
  • Lack of practical code-based examples in academic studies.

Justification for Proposed Research

This project bridges these gaps by combining theoretical usability principles with live interaction data and code-level implementation. The embedded feedback mechanism allows dynamic, continuous improvement aligned with user behavior and suggestions.

Evaluation Methods

  • Task Completion Rate: Percentage of users who completed a given task without assistance.
  • System Usability Scale (SUS): A standardized 10-question survey producing a usability score from 0–100.
  • Time on Task: The average duration users took to complete tasks.
  • Error Rate: Number of misclicks or incomplete interactions.
  • Real-time Feedback: Submitted through the integrated form.

Significance of the Project

The hybrid methodology combining heuristic evaluation and user feedback yields a holistic understanding of user interaction patterns.
